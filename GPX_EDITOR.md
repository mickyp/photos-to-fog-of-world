# GPX 軌跡編輯器 (GPX Editor)

> `scripts/gpx_editor.html` — 單一 HTML 檔案，直接用瀏覽器開啟，不需要 Server

## 功能概覽

這個工具是 `photos-to-fog-of-world` 專案的延伸功能：  
從照片產生的 GPX 軌跡點之間可能有斷點，這個編輯器讓你在地圖上可視化地補充這些缺失的路段，最後匯出一個密集的 GPX 軌跡（每 3 公尺一個點），適合匯入 Fog of World。

---

## 使用方式

直接在 Windows 檔案總管中找到 `scripts/gpx_editor.html`，雙擊用瀏覽器開啟即可。  
**不需要啟動任何 Server。**

---

## 操作流程

### 1. 載入 GPX
- 點選右上角「📂 載入 GPX」按鈕，或直接將 GPX 檔案拖曳到畫面上
- 原始 GPX 軌跡會以**紅色虛線**顯示（僅供參考）
- 所有 GPX 軌跡點自動轉為可編輯的**橘色編號節點**

### 2. 編輯節點（左側工具列）

| 工具 | 說明 |
|---|---|
| 🖐️ **平移** | 拖曳地圖（預設模式） |
| ➕ **新增** | 點地圖任意位置新增節點，自動插入到距離最近的路段中 |
| ✕ **刪除** | 點節點將其刪除 |
| 🛣️ **道路** | 選兩個節點，自動用道路路線填補中間路段 |
| ↩️ **復原** | 還原上一步（Ctrl+Z 也可以） |

> **節點上的數字**代表順序，方便在道路模式中確認要選擇哪兩個點。  
> **拖曳節點**可以調整位置，相鄰的道路段會自動重置為直線。

### 3. 道路吸附（🛣️ 道路模式）

進入道路模式後，頂部提示列會出現**交通方式選擇器**：

| 交通方式 | 路線顏色 | 適合場景 |
|---|---|---|
| 🚗 汽車/機車 | 🔵 藍色 | 一般道路、高速公路 |
| 🚲 腳踏車 | 🟢 綠色 | 自行車道、河濱路 |
| 🚶 步行 | 🩵 青色（預設） | 巷弄、步道 |
| ✏️ 直線 | ⬜ 灰色虛線 | 鐵路、渡輪、搭船等 |

操作步驟：
1. 選擇交通方式
2. 點選起點節點（變紫色）
3. 點選終點節點（可以跨越中間節點，中間節點會自動移除）
4. 等待路線計算（🚗🚲🚶 會呼叫 OSRM 免費公開 API，✏️ 直接畫直線）

### 4. 匯出 GPX
- 輸入軌跡名稱（留空會用預設名稱）
- 點「💾 匯出 GPX」
- 系統會：
  1. 將所有路段（道路路線 + 直線）整合成完整路徑  
  2. 每 3 公尺插入一個 GPX 軌跡點（Haversine 公式計算）  
  3. 根據首尾時間戳均等分配每個點的時間  
  4. 下載為 `<軌跡名稱>_fog_of_world_<時間戳>.gpx`

## 技術實作

| 功能 | 技術 |
|---|---|
| 地圖底圖 | [Leaflet.js](https://leafletjs.com/) + [CartoDB Voyager tiles](https://carto.com/basemaps/) |
| 道路路線 | [OSRM 公開 API](https://router.project-osrm.org/)（免費，支援 driving / cycling / foot） |
| 地理計算 | Haversine 公式，每 3m 插點 |
| 時間分配 | 根據累積距離線性內插於時間錨點之間 |
| 地圖互動 | 拖曳節點、點擊新增/刪除、道路吸附 |
| 語言切換 | 內建 i18n，繁體中文 / English |
| 匯出格式 | GPX 1.1，相容 Fog of World |

**所有功能純前端實作，無需後端，無需安裝任何軟體。**

---

## 注意事項

- 道路吸附（🚗🚲🚶）需要網路連線，才能呼叫 OSRM API
- OSRM 使用 OpenStreetMap 資料，台灣市區道路完整，偏遠地區或山路可能找不到路線（自動 fallback 為直線）
- 鐵路、渡輪等無法路由的段落，使用 ✏️ 直線模式
- 匯出的 GPX 若原始節點有時間戳記，會根據路段距離均等內插時間；若無時間戳記，則不加時間欄位
