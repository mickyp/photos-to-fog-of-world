# 版本規則

本專案採用三段式版本號：

`主版本.次版本.修訂版本`

例如：

- `1.0.0`
- `1.1.0`
- `1.1.1`

## 什麼時候升版

### 主版本

只有在有明顯不相容變更時才增加。

例如：

- 操作流程大幅改變
- 舊教學或舊參數不再適用
- 輸出格式或行為有破壞性修改

### 次版本

新增功能，但原本用法仍然可用時增加。

例如：

- 新增 GUI 功能
- 新增新的匯出選項
- 新增更多照片格式或處理能力

### 修訂版本

只有修 bug、穩定性改善、介面微調時增加。

例如：

- 修正路徑同步問題
- 修正視窗行為
- 修正打包或執行問題

## 版號來源

專案唯一的版號來源是根目錄的 `VERSION` 檔案。

釋出新版本時，只需要修改這個檔案。

## 版本顯示位置

Windows 打包輸出的檔名維持固定：

- `photos-to-gpx-cli.exe`
- `photos-to-gpx-gui.exe`

版號應顯示在程式內容中，而不是執行檔檔名：

- GUI：顯示在視窗標題
- CLI：可用 `--version` 查看

## 建議釋出流程

1. 修改根目錄的 `VERSION`
2. 執行 `py scripts/generate_changelog.py`
3. 檢查 `CHANGELOG.md`
4. 執行打包腳本
5. 建立對應的 release commit 與 tag
